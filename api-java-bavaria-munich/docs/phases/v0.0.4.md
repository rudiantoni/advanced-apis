# v0.0.4 (Phase 4): Postgres and ORM implementation replacing in-memory storage

Goals:

1. Replace the in-memory adapter with real persistence using Postgres via JPA ORM.
2. Create draft SQL files with the initial structure required for connection and database operations.

## API change

From this phase on, the API is connected to Postgres through JPA. Postgres must be running and reachable before starting the API; otherwise startup fails.

Another important detail is migrations. Migrations are `.sql` files whose code must be executed **once** directly on the **database**. Their counterparts are rollbacks, which undo all changes made by the migration.

From this phase on, run those migrations on the database before starting the API so the schema stays up to date. In this phase, for example, the `Product` entity maps to the Postgres table `public.product`. If it or any column is missing at API startup or during an operation, the operation fails and, depending on the error, the API may not start at all.

If the migration has never been run on the database configured in the project, execute the migration SQL (not the rollback) before running the API. After that, you can start and stop the API as often as you like while using the same database, because that migration does not need to run again (the `public.product` table already exists).

### `api-java-bavaria-munich/build.gradle`

Adds dependencies for JPA and the Postgres driver:

- [org.springframework.boot:spring-boot-starter-data-jpa](https://spring.io/projects/spring-data-jpa): [Maven Central artifact](https://central.sonatype.com/artifact/org.springframework.boot/spring-boot-starter-data-jpa)
- [org.postgresql:postgresql](https://jdbc.postgresql.org/): [Maven Central artifact](https://central.sonatype.com/artifact/org.postgresql/postgresql)

```gradle
dependencies {
    // ...
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
    runtimeOnly 'org.postgresql:postgresql'
}
```

### `api-java-bavaria-munich/src/main/resources/application.yml`

Configures JPA and the Postgres connection.

```yml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/adv_pg_db
    username: adv_pg_user
    password: adv_pg_pass
    driver-class-name: org.postgresql.Driver
  jpa:
    show-sql: false
    hibernate:
      ddl-auto: none
    properties:
      hibernate:
        format_sql: false
```

### `api-java-bavaria-munich/src/main/java/com/myapps/bavariamunich/entity/Product.java`

`Product` entity. Converted to a JPA entity instead of behaving only as a POJO.

```java
package com.myapps.bavariamunich.entity;

import javax.persistence.*;
import java.math.BigDecimal;

@Entity
@Table(schema = "public", name = "product")
public class Product {

    @Id
    @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "product_seq")
    @SequenceGenerator(name = "product_seq", sequenceName = "public.product_seq", allocationSize = 1)
    @Column(name = "id")
    private Long id;

    @Column(name = "name", nullable = false, length = 256)
    private String name;

    @Column(name = "description", nullable = false, length = 512)
    private String description;

    @Column(name = "price", nullable = false, precision = 19, scale = 2)
    private BigDecimal price;

    @Column(name = "reference", length = 128)
    private String reference;

    @Column(name = "stock_quantity")
    private Integer stockQuantity;

    @Column(name = "image_url", length = 1024)
    private String imageUrl;

    public Product() {
    }

    public Product(
            Long id,
            String name,
            String description,
            BigDecimal price,
            String reference,
            Integer stockQuantity,
            String imageUrl
    ) {
        this.id = id;
        this.name = name;
        this.description = description;
        this.price = price;
        this.reference = reference;
        this.stockQuantity = stockQuantity;
        this.imageUrl = imageUrl;
    }

    public Product copy() {
        return new Product(id, name, description, price, reference, stockQuantity, imageUrl);
    }

   // getters and setters
}
```

### `api-java-bavaria-munich/src/main/resources/draft-migration.sql`

Draft SQL migration file.

```sql
-- -------------------------------
-- Table public.product
-- -------------------------------
CREATE TABLE IF NOT EXISTS public.product (
    id                       BIGINT NOT NULL,
    name                     VARCHAR(256) NOT NULL,
    description              VARCHAR(512) NOT NULL,
    price                    NUMERIC(19, 2) NOT NULL,
    reference                VARCHAR(128),
    stock_quantity           INTEGER,
    image_url                VARCHAR(1024),
    CONSTRAINT pk_product    PRIMARY KEY (id)
);

CREATE SEQUENCE IF NOT EXISTS public.product_seq
    AS BIGINT
    INCREMENT 1
    MINVALUE 1
    NO MAXVALUE
    NO CYCLE
    START WITH 1
    OWNED BY public.product.id;

ALTER TABLE public.product
    ALTER COLUMN id SET DEFAULT nextval('public.product_seq'::regclass);
```

### `api-java-bavaria-munich/src/main/resources/draft-rollback.sql`

Draft SQL rollback for the migration.

```sql
-- -----------------------------------------
-- WARNING: THIS IS A DATABASE ROLLBACK SCRIPT!
-- -----------------------------------------
-- -------------------------------
-- Table public.product
-- -------------------------------
ALTER SEQUENCE IF EXISTS public.product_seq RESTART WITH 1;

DROP TABLE IF EXISTS public.product;
```
